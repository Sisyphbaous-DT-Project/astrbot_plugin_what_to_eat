# 是啊，吃什么

![插件图标](logo.png)

一款为 AstrBot 开发的简单有趣的插件，帮助大学生解决每天最头疼的问题——今天吃什么？

## 🎯 功能特点

- **智能识别**：自动识别消息中的"吃什么"关键词
- **随机推荐**：根据设置的概率随机推荐美食或复读"是啊，吃什么"
- **可编辑食物库**：内置50个大学生常吃的食物，支持自由增删改
- **自定义扩展**：可添加额外的自定义食物列表
- **概率配置**：灵活调整推荐 vs 复读的概率比例

## 📦 安装方法

### 方法一：WebUI 上传（推荐）

1. 打开 AstrBot 的 WebUI
2. 进入"插件市场"
3. 点击"上传插件"
4. 选择 `astrbot_plugin_what_to_eat.zip` 文件
5. 重启 AstrBot

### 方法二：手动安装

1. 将插件 ZIP 文件解压到 `data/plugins/shi_a_chi_shen_me/`
2. 重启 AstrBot
3. 在 WebUI 中启用插件

## ⚙️ 配置说明

在 WebUI → 插件管理 → 是啊，吃什么 → 配置 中可以进行以下设置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `recommend_probability` | float | 0.3 | 推荐食物的概率（0.0-1.0） |
| `builtin_foods` | list | 50个食物 | 内置食物列表，可自由编辑 |
| `custom_foods` | list | [] | 额外的自定义食物列表 |

### 配置示例

**推荐概率设为 0.5（一半概率推荐）：**
```json
{
  "recommend_probability": 0.5
}
```

**添加自定义食物：**
```json
{
  "custom_foods": ["海底捞", "肯德基", "麦当劳"]
}
```

**修改内置食物列表：**
直接在 `builtin_foods` 中添加、删除或修改食物名称

## 💬 使用示例

### 基本用法

用户发送包含"吃什么"的消息：

```
用户：今天中午吃什么好呢？
机器人：要不吃黄焖鸡米饭？

用户：晚上吃什么
机器人：是啊，吃什么

用户：@机器人 吃什么
机器人：试试麻辣烫吧！
```

### 触发示例

以下消息都会触发插件：
- "吃什么"
- "今天中午吃什么"
- "晚上吃什么好呢"
- "不知道吃什么"
- "@机器人 吃什么"

## 🎲 概率说明

- **推荐食物**（概率：P）：从食物列表中随机选择一个推荐
- **复读**（概率：1-P）：回复"是啊，吃什么"

默认概率为 0.3，即：
- 30% 概率推荐食物
- 70% 概率复读"是啊，吃什么"

## 📝 回复模板

### 推荐回复（随机选择）
- "要不吃{食物}？"
- "试试{食物}吧！"
- "{食物}怎么样？"
- "推荐你吃{食物}~"
- "今天吃{食物}吧！"
- "{食物}了解一下？"

### 复读回复（固定）
- "是啊，吃什么"

## 🔧 技术细节

- **触发方式**：正则表达式匹配 `吃什么`
- **不需要@**：直接发送包含关键词的消息即可触发
- **阻断LLM**：识别到关键词后自动阻止默认LLM回复
- **群组/私聊**：都支持
- **线程安全**：使用随机数生成器安全选择食物

## 🐛 故障排除

### 插件加载失败
1. 检查 ZIP 文件是否完整
2. 确认 AstrBot 版本支持
3. 查看日志中的错误信息

### 不触发回复
1. 确认插件已启用
2. 检查消息是否包含"吃什么"三个字
3. 查看是否有其他插件冲突

### 食物列表为空
- 确保 `builtin_foods` 或 `custom_foods` 至少有一个不为空
- 检查配置文件格式是否正确

### LLM仍然回复
- 确保插件已正确安装和启用
- 检查是否有其他插件干扰
